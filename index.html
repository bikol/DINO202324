Lista poleceń Git

  • Polecenie Git Log

    Polecenie git log "branch" --not "branch" jest używane do wyświetlenia historii commitów, które są obecne w gałęzi o nazwie "branch", ale nie są obecne w innej gałęzi o tej samej nazwie.

    Ta komenda jest przydatna do śledzenia zmian między gałęziami i identyfikowania commitów, które zostały dodane do jednej gałęzi, ale nie zostały jeszcze dołączone do drugiej.

git add . Komenda git add służy do stageowania zmian. Zwykle po add podaje się listę plików które chcemy stageować. Ale można też zamiast listy plików użyć kropki aby stageować wszystkie pliki w repozytorium, które mają zmiany oraz nie są ignorowane.
