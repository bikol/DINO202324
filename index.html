git checkout -b "branch" Komenda git checkout -b "branch" umożliwia stworzenie nowej gałęzi (brancha). Bezpośrednio po wykonaniu komendy następuje przełączenie na nową gałąź, na której można zacząć pracę. Komenda git checkout -b "branch" jest równoważna dwóm komendom, następującym po sobie: git branch "branch" (tworzy nowy branch), git checkout "branch" (przechodzi na zadany branch).
git add . Komenda git add służy do stageowania zmian. Zwykle po add podaje się listę plików które chcemy stageować. Ale można też zamiast listy plików użyć kropki aby dodać do następnego commita wszystkie pliki w repozytorium, które mają zmiany oraz nie są ignorowane.
git blame -C "filename" Komenda git add Służy do analizy historii zmian w pliku, identyfikowania, która linia została ostatnio zmodyfikowana i kto dokonał tych zmian. Dodatkowo, opcja -C pozwala na wykrywanie przeniesionych fragmentów kodu między plikami.
